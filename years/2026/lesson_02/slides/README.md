# Lesson 02 Slides

Canva is the source of truth for these slides.

## Focus

Develop Swift data modelling habits for app scenarios, moving from simple values into reusable structures and collection-based logic.

## Learning Outcomes

- Choose appropriate Swift types for app data.
- Compare arrays, sets, and dictionaries for different storage needs.
- Use optionals, enums, structs, and classes in small examples.
- Explain value vs reference behavior using practical code.
- Write simple functions that create, update, filter, and summarize data.

## Rough Deck Flow

1. Recap of Lesson 01 Swift basics.
2. Why app features start with data modelling.
3. Collections: arrays, sets, and dictionaries.
4. Optionals and enums for real app states.
5. Structs, classes, and value vs reference behavior.
6. Functions for data manipulation.
7. Practice: model a small app domain and write helper functions.
8. Summary and preview of SwiftUI interfaces in Lesson 03.

## Demo And Practice Direction

Use public, generic app scenarios such as bookings, tasks, scores, or profiles. Keep examples practice-style and avoid reproducing private assessment scenarios or marking criteria.

## Assessment Preparation

This lesson prepares students to reason about data objects and data manipulation for TMA01. Keep all assessment alignment at the topic level.
