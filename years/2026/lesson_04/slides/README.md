# Lesson 04 Slides

Canva is the source of truth for these slides.

## Focus

Begin the post-TMA01 app-building arc with a multi-screen Blackjack app structure, shared state, navigation, reusable data objects, and project organization.

## Learning Outcomes

- Break a small game app into clear screens and responsibilities.
- Model cards, decks, players, hands, and game state as reusable Swift data objects.
- Use shared observable state for app-wide data.
- Build navigation between menu, table, and result-style screens.
- Organize a small Xcode project so code remains readable and extendable.

## Rough Deck Flow

1. Debrief the transition from TMA01 skills to app-building skills.
2. Blackjack as a public teaching domain for data, state, and UI flow.
3. Data objects for cards, decks, players, and game state.
4. Shared state and view updates in SwiftUI.
5. Navigation patterns for menu, gameplay, and results.
6. Project organization and naming conventions.
7. Practice: scaffold a multi-screen Blackjack app.
8. Summary and preview of interaction polish in Lesson 05.

## Demo And Practice Direction

Use a simplified Blackjack app as the running public example. Keep the game rules intentionally modest so students can focus on data flow, screen structure, naming, and maintainable project layout.

## Assessment Preparation

This lesson starts the TMA02 preparation arc by connecting data design, UI workflow, navigation, and project organization.
