# MTD 367 2026

This folder is for the 2026 public course run.

For 2026 onward, Canva is the source of truth for slides. Each lesson keeps Canva references and slide notes in `slides/README.md`, with student-facing exports in `slides/exports/`.

Private assessment material does not belong in this folder.

## Teaching Arc

The 2026 slide roadmap is assessment-aware but public-safe. Lesson materials should prepare students for the expected skills and workflows without reproducing private assessment questions, marking guides, rubrics, model answers, or private links.

TMA01 is expected immediately after Lesson 03, so Lessons 01-03 build from Swift fundamentals into simple SwiftUI interfaces and UI/UX basics. TMA02 is expected immediately after Lesson 06, so Lessons 04-06 develop a multi-screen Blackjack app with game logic, interaction polish, media feedback, navigation, and project organization. The ECA is expected roughly one month after Lesson 06 and should be supported by public learning themes around MapKit/location, reusable data objects, and location-based gameplay design.

AR is not part of the planned 2026 slide roadmap unless it is explicitly reintroduced later.

## Lesson Roadmap

| Lesson | Focus | Assessment Preparation |
| --- | --- | --- |
| Lesson 01 | Course orientation, Xcode setup, Swift basics, variables/constants, types, functions, conditionals, loops, and small runnable exercises. | Builds core Swift confidence for TMA01. |
| Lesson 02 | Swift data modelling for apps, collections, optionals, enums, structs/classes, value vs reference behavior, and data-manipulation functions. | Prepares students to explain and implement data objects and manipulation logic for TMA01. |
| Lesson 03 | SwiftUI foundations plus simple UI/UX fundamentals: views, state, bindings, forms, inputs, layout stacks, readability, hierarchy, spacing, mobile input flow, and parent-child data flow. | Final TMA01 preparation. |
| Lesson 04 | Multi-screen Blackjack app structure: shared state, navigation, menu/table/result flows, reusable data objects, and project organization. | Starts the TMA02 app-building arc. |
| Lesson 05 | Interaction polish for Blackjack: gestures, animation, visual feedback, audio feedback, usability tradeoffs, and gameplay-focused interaction choices. | Strengthens TMA02 interaction and presentation requirements. |
| Lesson 06 | Integrated Blackjack logic, navigation, media feedback, MapKit/location foundations, and location-based boss-hunt layering over Blackjack gameplay. | Final TMA02 preparation and public-safe ECA seeding. |

## Public Output Rules

- Keep Canva view/edit placeholders in each lesson's `slides/README.md`; do not publish private Canva links.
- Store student-facing exports only in each lesson's `slides/exports/` folder.
- Keep slide notes public-facing and practice-oriented.
- Use private assessment context only to derive topic coverage and readiness themes.
- Before publishing or committing slide material, check that no private assessment text, marking criteria, model answers, screenshots, or private links were added.
