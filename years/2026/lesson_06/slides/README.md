# Lesson 06 Slides

Canva is the source of truth for these slides.

## Focus

Bring the Blackjack app together for TMA02 readiness, then introduce MapKit/location as the public advanced-framework bridge toward a location-based Blackjack Boss Hunt experience.

## Learning Outcomes

- Integrate Blackjack game logic, navigation, interface state, and feedback into a coherent app.
- Explain hit, stand, bust, outcome, and restart flows in a simplified card-game implementation.
- Review code quality, naming, file organization, and project submission readiness.
- Describe MapKit/location foundations at a level suitable for a simple location-based game layer.
- Connect reusable data objects and UI design decisions to a map-based gameplay concept.

## Rough Deck Flow

1. Recap of Lessons 04-05 and the TMA02 skill stack.
2. Integrated Blackjack logic and gameplay states.
3. Navigation, feedback, and final usability review.
4. Code quality and project organization checklist.
5. TMA02 readiness review using public learning outcomes only.
6. MapKit/location foundations: maps, annotations, user location, and permission-aware design.
7. Public-safe ECA bridge: layering location-based boss encounters over Blackjack gameplay.
8. Course wrap-up and next-step practice guidance.

## Demo And Practice Direction

Use the simplified Blackjack app to review logic and project quality, then show a small MapKit/location demo that places public sample points on a map. Keep the boss-hunt discussion conceptual and public-safe: data objects, map annotations, navigation, and encounter flow.

## Assessment Preparation

This is the final preparation lesson before TMA02 and the public bridge toward the ECA. Cover Blackjack app design, UI workflow, gestures, animation/audio, game logic, navigation, code quality, project organization, MapKit/location foundations, and reusable data-object thinking. AR is not part of this roadmap.
