# Lesson 03 Slides

Canva is the source of truth for these slides.

## Focus

Introduce SwiftUI foundations and simple UI/UX design fundamentals so students can connect data, state, layout, and mobile input flow before TMA01.

## Learning Outcomes

- Build simple SwiftUI views with text, controls, stacks, and forms.
- Use `@State` and `@Binding` to connect interface changes to data.
- Describe parent-child data flow in a small SwiftUI interface.
- Apply basic UI/UX principles: readability, hierarchy, spacing, alignment, labels, and predictable input flow.
- Explain how interface choices affect mobile usability.

## Rough Deck Flow

1. Recap of Swift data modelling from Lesson 02.
2. Declarative UI and the role of a SwiftUI `View`.
3. Layout with stacks, forms, and common input controls.
4. Local state, bindings, and simple parent-child data flow.
5. UI/UX fundamentals for mobile screens.
6. Practice: design and implement a small data-entry interface.
7. TMA01 readiness review using public learning outcomes only.
8. Summary and preview of multi-screen apps in Lesson 04.

## Demo And Practice Direction

Use a simple public app form or calculator-style interface that demonstrates data entry, validation-friendly layout, computed display values, and a clear action button. Include lightweight critique prompts such as readability, spacing, hierarchy, and input order.

## Assessment Preparation

This is the final preparation lesson before TMA01. Cover Swift basics, data objects, data manipulation, SwiftUI state/binding, and simple UI/UX fundamentals without including private assessment text.
