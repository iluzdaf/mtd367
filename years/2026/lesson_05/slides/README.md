# Lesson 05 Slides

Canva is the source of truth for these slides.

## Focus

Polish the Blackjack app with gestures, animation, visual feedback, audio feedback, and usability decisions that support gameplay.

## Learning Outcomes

- Add tap, long-press, and swipe-style interactions where they fit user intent.
- Use animation to clarify state changes rather than decorate randomly.
- Add audio or media feedback for meaningful gameplay moments.
- Evaluate usability tradeoffs for gesture-based controls.
- Improve a game interface through consistent visual feedback and interaction design.

## Rough Deck Flow

1. Recap of the multi-screen Blackjack structure from Lesson 04.
2. Choosing controls and gestures for gameplay actions.
3. Gesture usability: discoverability, accessibility, and fallback controls.
4. Animation for dealing, revealing, outcomes, and state changes.
5. Audio and visual feedback for player confidence.
6. Practice: add interaction polish to a Blackjack screen.
7. TMA02 readiness checkpoint for interaction and presentation.
8. Summary and preview of MapKit/location foundations in Lesson 06.

## Demo And Practice Direction

Use the Blackjack teaching app from Lesson 04. Demonstrate one gesture, one animation, and one feedback pattern at a time, then ask students to explain how each choice helps or hurts the player experience.

## Assessment Preparation

This lesson strengthens TMA02 readiness around interaction design, gestures, animation/audio, usability reasoning, and polished app presentation.
