_notes: Be safer than Objective-C Swift helps catch bugs at compile time rather than runtime, and eliminates many risky programming practices that were common in Objective-C. - Optionals make nil handling explicit - Strong static typing - Automatic memory safety - Type-safe collections - Immutable by default Be faster than interpreted languages Swift compiles to machine code, whereas interpreted languages run code line by line at runtime. Compiled code is much faster. Swift uses LLVM, the same compiler infrastructure used for C, C++ and Rust. - Compiles to machine code - No runtime interpreter needed - LLVM optimizations - Static typing = fewer runtime checks - Efficient memory management
